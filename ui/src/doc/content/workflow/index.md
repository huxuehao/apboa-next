# 工作流管理

工作流是智能体业务编排的核心引擎，通过可视化拖拽将开始、结束、逻辑判断、数据处理等节点自由组合，灵活接入缓存、数据源、消息队列等外部资源，实现节点间变量的精准传递与条件路由。调试运行让每一步执行状态一目了然，发布后生成不可变版本并绑定路由，将复杂的业务过程沉淀为可调用、可追踪、可复用的标准化流程。


## 功能概述

- 可视化画布拖拽编辑，支持节点连线、自动布局
- 丰富的节点类型：基础节点、逻辑节点、数据节点、集成节点等
- 支持节点间变量传递与数据绑定
- 支持条件分支、循环、迭代等控制流
- 集成缓存、数据库、消息队列等外部资源
- 调试运行与执行结果可视化
- 版本管理与发布机制
- 编辑锁与协作控制
- 全局变量定义与引用


## 核心概念

### 工作流状态

| 状态 | 说明 |
|------|------|
| **草稿（DRAFT）** | 工作流处于编辑状态，可以进行修改 |
| **已发布（PUBLISHED）** | 工作流已发布为不可变版本，可被智能体调用 |

### 节点类型总览

工作流支持多种节点类型，按功能分为以下几类：

| 分类 | 节点类型 | 数量 |
|------|----------|------|
| 基础节点 | 开始节点、结束节点 | 2 |
| 逻辑节点 | 条件分支、迭代处理、循环、非空选择、结果匹配 | 5 |
| 数据节点 | 数据库CRUD、缓存操作、消息队列 | 10 |
| 集成节点 | 智能体、工具执行、MCP调用、HTTP请求、代码执行 | 5 |
| 转换节点 | 字符串分割、字符串模板、序列化、反序列化 | 4 |
| 列表节点 | 列表过滤、列表排序 | 2 |
| 变量节点 | 变量聚合 | 1 |


## 节点配置详解

### 基础节点

#### 一、开始节点（START）

开始节点是工作流的入口，用于定义工作流的请求参数。

**配置面板说明：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 节点名称 | 节点的显示名称 | 是 |
| 参数列表 | 定义工作流的输入参数 | 否 |

**参数配置：**

点击"添加"按钮可以添加参数，每个参数包含以下属性：

| 属性 | 说明 |
|------|------|
| 参数名称 | 参数的标识符（不允许包含下划线 `_`） |
| 参数类型 | 支持：String、Long、Integer、Float、Double、Boolean、Array、Object |
| 是否必填 | 标记参数是否为必填项 |
| 默认值 | 参数的默认值（可选） |

**输出说明：**

开始节点的输出即为定义的参数列表，每个参数都会成为可被下游节点引用的输出。

#### 二、结束节点（END）

结束节点是工作流的出口，用于生成最终响应。

**配置面板说明：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 节点名称 | 节点的显示名称 | 是 |
| 输入绑定 | 绑定上游节点的输出作为输入 | 是 |
| 响应模板 | 定义最终输出的内容模板 | 是 |
| 响应模板格式 | 模板的解析格式 | 是 |

**输入绑定：**

选择一个上游节点的输出作为结束节点的输入，通常绑定为 `input`。

**响应模板：**

使用模板语法定义输出内容，支持以下格式：

| 格式 | 说明 | 示例 |
|------|------|------|
| 普通字符串 | 直接使用 `${}` 引用变量 | `${input}` |
| Jackson JSON | 保持 JSON 类型 | `${input}` |
| Velocity 模板 | 支持条件判断和循环 | `#if($input.status == "success") 成功 #else 失败 #end` |

**常用模板示例：**

```
# 直接返回输入值
${input}

# 返回 JSON 格式
{"result": ${input}, "status": "success"}

# 使用 Velocity 条件判断
#if($input.code == 200)
  操作成功：$input.message
#else
  操作失败：$input.message
#end
```

### 逻辑节点

#### 三、条件分支节点（IF_ELSE）

条件分支节点用于根据条件选择执行路径，类似编程语言中的 if-else 语句。

**配置面板说明：**

条件分支节点的配置面板采用 IF-THEN-ELSE 三段式布局：

| 配置区域 | 说明 |
|----------|------|
| IF 区域 | 配置条件判断逻辑 |
| THEN 区域 | 配置条件成立时的跳转节点 |
| ELSE 区域 | 配置条件不成立时的跳转节点 |

**IF 区域配置项：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 输入绑定 | 绑定需要判断的值 | 是 |
| 关系运算符 | 比较运算符 | 是 |
| 比较值 | 常量值或节点输出 | 是（表达式模式除外） |
| 条件表达式 | Groovy 表达式（仅表达式模式） | 是（表达式模式） |

**关系运算符列表：**

| 运算符 | 说明 |
|--------|------|
| EQ | 等于（==） |
| NE | 不等于（!=） |
| GT | 大于（>） |
| LT | 小于（<） |
| GE | 大于等于（>=） |
| LE | 小于等于（<=） |
| CONTAINS | 包含 |
| NOT_CONTAINS | 不包含 |
| STARTS_WITH | 开头匹配 |
| ENDS_WITH | 结尾匹配 |
| EQUALS | 严格等于（区分大小写） |
| NOT_EQUALS | 严格不等于（区分大小写） |
| IS_TRUE | 为 true |
| IS_FALSE | 为 false |
| EXPRESSION | 表达式模式 |

**比较值类型：**

| 类型 | 说明 |
|------|------|
| 常量 | 直接输入固定值进行比较 |
| 节点输出 | 选择上游节点的输出值进行比较 |

**高级选项：**

| 配置项 | 说明 |
|--------|------|
| 计算对象 | 值本身（SELF）或长度（LENGTH） |
| 输入为空时视为True | 当输入值为 null 时是否视为条件成立 |

**THEN/ELSE 区域配置项：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 跳转到 | 选择条件成立/不成立时的下一个节点 | 是 |

#### 四、迭代处理节点（ITERATE）

迭代处理节点用于对集合输入逐项执行迭代处理代码。

**配置面板说明：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 节点名称 | 节点的显示名称 | 是 |
| 输入绑定 | 绑定需要迭代的集合数据 | 是 |
| 语言 | 代码语言（目前支持 Java） | 是 |
| 迭代处理代码 | 处理每个元素的代码 | 是 |

**输入绑定：**

通常绑定一个数组或集合作为迭代数据源。

**迭代处理代码示例：**

```java
package com.hxh.apboa.node.iterate.load;

import java.util.*;
import com.hxh.apboa.node.iterate.IteratorExecutor;

/**
 * 数据加载处理实现类。
 * 作为迭代器执行器，对可迭代对象中的每个元素进行处理。
 */
public class DataProcess implements IteratorExecutor {

    /**
     * 处理迭代过程中的单个元素
     *
     * @param item  当前迭代到的子元素
     * @param index 当前子元素在可迭代对象中的位置索引
     * @return 处理后的子元素对象
     */
    @Override
    public Object doIterate(Object item, Integer index) {
        // 对item进行处理
        return item;
    }
}
```

#### 五、循环节点（LOOP）

循环节点用于按次数或数据源重复执行子工作流。

**配置面板说明：**

| 配置区域 | 说明 |
|----------|------|
| 节点名称 | 节点的显示名称 |
| 输入绑定 | 绑定上游节点输出 |
| 迭代设置 | 配置循环模式和参数 |
| 子流程配置 | 定义每次迭代执行的处理逻辑 |

**迭代设置配置项：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 循环模式 | 数据迭代或计数循环 | 是 |
| 迭代数据源 | 选择输入绑定中提供可迭代数据的变量（数据迭代模式） | 是（数据迭代模式） |
| 元素变量名 | 子流程中访问当前元素的变量名 | 是 |
| 循环索引变量名 | 子流程中访问当前索引的变量名 | 是 |
| 最大循环次数 | 防止死循环的安全上限 | 是 |

**循环模式说明：**

| 模式 | 说明 |
|------|------|
| 数据迭代 | 遍历输入绑定提供的数据（数组或集合），逐项执行子流程 |
| 计数循环 | 按固定次数执行子流程，不依赖外部数据 |

**子流程配置：**

1. 点击"进入子流程可视化编辑"按钮进入子流程编辑模式
2. 在子流程中添加和配置节点
3. 选择入口节点（子流程中没有入边的节点）
4. 完成编辑后返回主流程

**终止表达式（高级选项）：**

Groovy 表达式，求值为 true 时终止循环。可使用循环变量名、元素变量名和主流程全局变量。

示例：
- `item == null` - 当元素为空时终止
- `loopIndex >= 10` - 当索引大于等于 10 时终止

#### 六、非空选择节点（NON_EMPTY_SELECT）

非空选择节点用于从多个候选输入中选择第一个或最后一个非空值。

**配置面板说明：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 节点名称 | 节点的显示名称 | 是 |
| 输入绑定 | 绑定多个候选值（支持拖拽排序） | 是 |
| 选择策略 | 第一个或最后一个非空值 | 是 |
| 默认选择 | 所有输入都为空时的默认跳转节点 | 否 |

**选择策略：**

| 策略 | 说明 |
|------|------|
| 第一个 | 按输入绑定顺序返回第一个非空值 |
| 最后一个 | 遍历所有输入，返回最后一个非空值 |

**使用场景：**

- 从多个数据源中获取数据，使用第一个成功返回的结果
- 合并多个可能为空的字段，取第一个有值的

#### 七、结果匹配节点（MATCH_RESULT）

结果匹配节点用于按匹配值选择不同后续节点，类似 switch-case 语句。

**配置面板说明：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 节点名称 | 节点的显示名称 | 是 |
| 输入绑定 | 绑定需要匹配的值 | 是 |
| 匹配类型 | 等于或包含 | 是 |
| 匹配规则列表 | 定义匹配值和对应的跳转节点 | 是 |
| 区分大小写 | 是否区分大小写进行匹配 | 否 |
| 默认输出 | 所有条件都不满足时的跳转节点 | 否 |

**匹配类型：**

| 类型 | 说明 |
|------|------|
| 等于 | 输入值完全等于匹配值 |
| 包含 | 输入值包含匹配值 |

**匹配规则配置：**

每条匹配规则包含：
- 匹配值：要匹配的值（常量或节点输出）

### 数据节点

#### 八、数据库查询节点（DB_SELECT）

数据库查询节点用于执行 SQL 查询操作。

**配置面板说明：**

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| 节点名称 | 节点的显示名称 | 是 |
| 输入绑定 | 绑定 SQL 参数 | 否 |
| 数据源 | 选择已配置的数据源 | 是 |
| SQL 语句 | 编写查询 SQL | 是 |
| 参数绑定 | SQL 参数值列表 | 否 |
| 参数模板格式 | 参数的解析格式 | 是 |

**SQL 语句编写规范：**

- 使用 `?` 作为参数占位符
- 参数按顺序一一对应
- 支持 Velocity 模板语法

**SQL 示例：**

```sql
-- 简单查询
SELECT * FROM users WHERE id = ?

-- 多参数查询
SELECT * FROM orders WHERE user_id = ? AND status = ? AND create_time > ?

-- 使用 Velocity 模板
SELECT * FROM users WHERE 1=1
#if($userName)
  AND name LIKE '%${userName}%'
#end
#if($status)
  AND status = '${status}'
#end
```

**参数绑定配置：**

每个参数包含：

| 属性 | 说明 |
|------|------|
| 参数值 | 参数的值，支持 `${输入绑定名}` 引用 |
| 参数类型 | STRING、INTEGER、INT、LONG、DOUBLE、FLOAT、BOOLEAN、BOOL |

**参数模板格式：**

| 格式 | 说明 |
|------|------|
| 纯文本替换 | 直接替换变量，结果为字符串 |
| JSON 保类型 | 保持 JSON 数据类型 |
| Velocity 模板 | 支持 Velocity 模板语法 |

#### 十七、智能体节点（AGENT）

智能体节点用于调用 AI 模型进行推理。

- **连线**：从节点输出端口拖拽到另一个节点的输入端口
- **删除节点**：选中节点后按 Delete 键，或右键菜单删除

### 变量与数据绑定

#### 输入绑定

节点的输入可以绑定为：

- **固定值**：直接输入常量值
- **节点输出**：引用其他节点的输出值
   - 格式：`节点ID.输出名称`
   - 例如：`start.userId` 表示开始节点的 userId 输出

#### 快捷输入绑定名

在配置节点参数时，可以通过**双击 Ctrl 键**快速插入输入绑定名称：

1. 将光标定位到需要插入绑定名的位置
2. 快速按两次 Ctrl 键
3. 弹出输入绑定选择列表
4. 选择需要的绑定名称即可插入

:::tip
快捷输入功能可以大幅提高配置效率，避免手动输入长绑定名称。
:::

#### 全局变量

工作流支持定义全局变量，在所有节点中都可以引用：

1. 点击左侧工具栏的"变量"按钮
2. 在变量面板中添加全局变量
3. 设置变量名称、类型和默认值
4. 在节点配置中通过 `${变量名}` 引用

:::warning
全局变量名称在同一工作流中必须唯一，建议使用有意义的名称便于理解。
:::


### 参数模板格式

部分节点支持参数模板配置，有两种格式可选：

#### STRING 格式

直接使用字符串，支持 `${}` 变量引用：

```
Hello, ${name}! Your order ${orderId} is ready.
```

#### VELOCITY 格式

使用 Apache Velocity 模板语法，支持更复杂的逻辑：

```velocity
#if($age >= 18)
  您已成年
#else
  您未成年
#end

#foreach($item in $list)
  - $item.name: $item.value
#end
```

:::tip
大多数场景下使用 STRING 格式即可，需要条件判断或循环处理时使用 VELOCITY 格式。
:::


### 调试与运行

#### 调试运行

1. 点击顶部"调试"按钮打开调试面板
2. 在输入框中填写 JSON 格式的请求参数：
   ```json
   {
     "params": [
       { "name": "userId", "value": "123" },
       { "name": "userName", "value": "张三" }
     ],
     "variables": {
       "env": "test"
     }
   }
   ```
3. 点击"运行"按钮执行工作流
4. 查看执行结果和每个节点的执行状态

#### 执行结果

调试运行后可以查看：

- **整体状态**：成功或失败
- **节点执行详情**：每个节点的执行状态、输入输出、耗时
- **错误信息**：执行失败时的错误详情

#### 节点状态标识

| 状态 | 说明 |
|------|------|
| 空闲（IDLE） | 节点未执行 |
| 运行中（RUNNING） | 节点正在执行 |
| 成功（SUCCESS） | 节点执行成功 |
| 失败（FAILED） | 节点执行失败 |
| 未校验（UNVERIFIED） | 节点配置校验未通过 |


### 版本管理

#### 发布工作流

1. 点击"校验"按钮验证工作流配置
2. 校验通过后点击"发布"按钮
3. 填写版本备注（可选）
4. 确认发布

:::warning
发布后的工作流为不可变版本，无法直接修改。如需修改，请创建新版本或复制工作流。
:::

#### 版本历史

点击"版本"按钮可以查看：

- 版本列表
- 版本详情
- 加载历史版本


### 编辑锁

#### 锁定机制

- 同一时间只能有一个用户编辑工作流
- 进入编辑器时自动获取编辑锁
- 退出编辑器时自动释放编辑锁
- 如果工作流已被其他用户锁定，将进入只读模式

#### 只读模式

在只读模式下：

- 无法添加、删除、修改节点
- 无法保存工作流
- 无法调试运行
- 可以查看工作流配置


## 资源管理

### 数据源

在工作流管理页面点击"资源维护"按钮，可以管理数据源：

- **添加数据源**：配置数据库连接信息
- **启用/禁用**：控制数据源是否可用
- **编辑/删除**：修改或删除数据源

### 缓存

管理 Redis 缓存实例：

- **添加缓存**：配置 Redis 连接信息
- **启用/禁用**：控制缓存是否可用

### 消息队列

管理消息队列实例：

- **添加消息队列**：配置 Kafka、RabbitMQ 或 RocketMQ 连接信息
- **启用/禁用**：控制消息队列是否可用


## 与智能体集成

### 在智能体中使用工作流

1. 进入智能体配置页面
2. 在"工具与能力"步骤中选择"工作流"
3. 选择已发布的工作流
4. 配置工作流输入参数

### 工作流调用方式

智能体调用工作流时：

- 自动传递配置的输入参数
- 获取工作流执行结果作为输出
- 支持同步调用，等待工作流执行完成


## 最佳实践

### 节点命名规范

- 使用有意义的名称，便于理解节点功能
- 避免使用默认名称，如"节点1"、"节点2"
- 建议格式：`动作_对象`，如`查询_用户信息`、`发送_通知`

### 错误处理

- 在关键节点后添加条件分支，处理异常情况
- 使用结束节点返回友好的错误信息
- 在调试阶段仔细检查每个节点的执行结果

### 性能优化

- 避免不必要的节点，减少执行路径
- 合理使用并行分支，提高执行效率
- 对于耗时操作，考虑使用异步处理

### 数据安全

- 敏感信息使用变量引用，避免硬编码
- 配置适当的参数校验，防止注入攻击
- 定期审查工作流配置，确保安全性


## 常见问题

### 工作流无法保存？

- 检查是否已获取编辑锁
- 确认是否有未通过的配置校验
- 查看网络连接是否正常

### 调试运行失败？

- 检查请求参数格式是否正确
- 确认数据源、缓存等资源是否已配置并启用
- 查看错误信息，定位具体失败节点

### 节点配置校验不通过？

- 检查必填项是否已填写
- 确认参数格式是否正确
- 查看校验错误提示，按提示修改

### 工作流被锁定无法编辑？

- 工作流可能被其他用户编辑中
- 联系锁定用户释放锁，或等待锁超时
- 可以复制工作流进行编辑

### 如何回滚到历史版本？

- 点击"版本"按钮查看版本历史
- 选择需要回滚的版本
- 点击"加载"恢复到该版本

### 节点输出如何传递给下一个节点？

- 在下一个节点的输入配置中，选择"节点输出"类型
- 选择源节点和输出名称
- 系统会自动在运行时传递数据

### 循环节点如何使用？

1. 添加循环节点
2. 配置循环模式（按次数或按数据源）
3. 点击"编辑子流程"进入子流程编辑
4. 在子流程中添加需要重复执行的节点
5. 完成编辑后返回主流程

### 条件分支如何配置多个条件？

- 使用"结果匹配"节点实现多分支
- 或者嵌套使用"条件分支"节点
- 每个分支可以连接不同的后续节点

### 如何使用快捷输入功能？

- 在节点配置的输入框中，快速按两次 Ctrl 键
- 弹出绑定选择列表，选择需要的绑定名称
- 该功能可以快速插入节点输出引用，避免手动输入

### 全局变量如何定义和使用？

1. 点击左侧工具栏的"变量"按钮打开变量面板
2. 点击"添加"按钮创建全局变量
3. 设置变量名称、类型和默认值
4. 在节点配置中使用 `${变量名}` 引用
